# TSVコード体系定義書 (TSV Code Specification)

## 0. 主文
TSVコードは、Webアプリケーション「ShimawaNever」における書籍、書類、発行者、及び書架の識別及び管理の用に供する識別子規格であって、TSVコード並びにTSVコードの子体系として制定されるS-TSVコード、S-TSV 検証コード、及びS-TSV一次元シンボルの構造、生成規則、及び運用に関する事項は、本ドキュメントに定めるところによる。
本規格は、書籍、書類、発行者、及び書架に対して一意の識別子を付与することにより、オブジェクトの追跡、管理、及び相互参照を可能とすることを目的とする。
本ドキュメントは、TSVコードを用いるすべてのシステム（以下、システムという。）に対し、本ドキュメントに定めるところに従ってTSVコード、S-TSVコード、S-TSV検証コード、及びS-TSV一次元シンボルを生成し、管理し、運用することを求める。

## 1. コード構造 (Code Structure)
TSVコードは、これより先に示す構成要素からなるコード体系である。

### 1.1. オブジェクト
システムは、TSVコードによって識別されるそれぞれの書籍、書類、発行者、及び書架を、1つのオブジェクトとして取り扱う。
TSVコードの割り当てを受けるオブジェクトは、次の番号及び記号の割り当てを受けなければならない。また、これらの番号及び記号は、本ドキュメントに定めるところに従って割り当てる。
1. オブジェクト区分記号
2. オブジェクト区分コード
3. 固有番号

### 1.2. 完全記法 TSVコード (Full Notation / TSV Code)
TSVコードは、18桁の英数字及びハイフンで構成される文字列であって、本コード体系における完全記法の識別子である。
システムは、各オブジェクトに対して本記法に基づくTSVコードを生成し、これらを主たる識別子として割り当てる。
TSVコードの構造は次に示すとおりである。

- **フォーマット:** `TSV-[TT]-[2SS]-[XXXXXXXXX]-[C]`
- **構成:**
    - `TSV`: システム識別子（固定値）
    - `TT`: オブジェクト区分記号（2文字）
    - `2SS`: オブジェクト区分コード（3桁）
    - `XXXXXXXXX`: 固有番号（9桁）
    - `C`: チェック数字（1桁）

### 1.3. 検証用数字 S-TSV検証コード (Validation Number / S-TSV Validation Code)
S-TSV検証コードは、TSVコード及びS-TSVコードに関するチェック数字の算出の用に供する12桁の数字列である。
システムは、オブジェクトに対してTSVコードを割り当てるとき、本記法に基づくS-TSV検証コードを生成し、当該コードによって当該オブジェクトのTSVコードのチェック数字を算出し、当該チェック数字を当該オブジェクトのTSVコードの末尾に付与する。
S-TSV検証コードの構造は次に示すとおりである。

- **フォーマット:** `[2SS][XXXXXXXXX]`（計12桁）
- **構成:** オブジェクト区分コード(3) + 固有番号(9)

### 1.4. 短縮記法 S-TSVコード (Short Notation / S-TSV Code)
S-TSVコードは、13桁の数字列であって、本コード体系における短縮記法の識別子である。
システムは、貼付用S-TSV一次元シンボルを発行するとき、当該オブジェクトに対して本記法に基づくS-TSVコードを生成する。但し、本ドキュメントは、S-TSVコードのデータベース内保管を強制するものではない。
S-TSVコードの構造は次に示すとおりである。

- **フォーマット:** `[2SS][XXXXXXXXX][C]`（計13桁）
- **構成:** オブジェクト区分コード(3) + 固有番号(9) + チェック数字(1)

### 1.5. 短縮記法に基づく管理コード S-TSV一次元シンボル (Short Notation-based Management Code / S-TSV One-Dimensional Symbol)
S-TSV一次元シンボルは、本コード体系によって管理されるオブジェクトに貼付されるGTIN-13（JAN-13/EAN-13）形式の一次元シンボルである。
システムは、物理オブジェクトに対する管理コードの発行の請求を受けたとき、当該オブジェクトのS-TSVコードを内容とするS-TSV一次元シンボルを生成し、当該S-TSV一次元シンボルを当該オブジェクトに貼付するための管理コードとして発行する。

## 2. オブジェクト区分に基づくオブジェクト区分記号、オブジェクト区分コード、及び固有番号 (Object Classification-based Object Classification Symbols, Object Classification Codes, and Unique Number Construction)
TSVコードの割り当てを受けるオブジェクトは、これより先に示すオブジェクト区分に属し、各オブジェクト区分ごとに定められたオブジェクト区分記号、オブジェクト区分コードと、区分ごとに定められた生成規則に従って生成された固有番号の割り当てを受けるものとする。

### 2.1. オブジェクト区分
TSVコードの割り当てを受けるオブジェクトは、次に示すいずれかのオブジェクト区分に属するものとする。
システムは、TSVコードの割り当てを受けるオブジェクトに対し、これより先に定めるところに従って、当該オブジェクトにオブジェクト区分記号、オブジェクト区分コード、及び固有番号の割り当てを行う。

| オブジェクト区分 | 名称 | 説明 |
| :--- | :--- | :--- |
| **TSV-BK オブジェクト** | 書籍 | オブジェクトのうち、専ら書籍としての性質を有するもの。 |
| **TSV-BG オブジェクト** | 書籍グループ | TSV-BK オブジェクトをグループ化するためのオブジェクト。書籍のシリーズ（前後編に分されるもの、定期刊行物、雑誌扱いのうち複数巻に分かれるものなど）を表す。 |
| **TSV-RE オブジェクト** | 記録資料等 | オブジェクトのうち、書籍以外の記録資料、文書、その他の物品を表すもの。 |
| **TSV-IS オブジェクト** | 資料発行者 | TSV-RE オブジェクトの発行者を表すオブジェクト。TSV-RE オブジェクトは、必ず1つの TSV-IS オブジェクトと関連付けられる。 |
| **TSV-SH オブジェクト** | 書架 | 書籍や書類を収納するための物理的な構造物を表すオブジェクト。書架は、親子構造を持つことができる。 |

### 2.2. オブジェクト区分とオブジェクト区分記号、オブジェクト区分コード、及び固有番号
オブジェクト区分に基づくオブジェクト区分記号及びオブジェクト区分コードは、これより先に定めるところによる。
オブジェクト区分に基づく固有番号の生成規則は、これより先に定めるところによる。

#### 2.2.1. TSV-BK オブジェクト（書籍）
| オブジェクト区分 | オブジェクト区分の名称 | オブジェクト区分記号 | オブジェクト区分コード |
| :---: | :---: | :---: | :---: |
| **TSV-BK オブジェクト** | 書籍 | BK | 200 - 202 |

TSV-BK オブジェクト「書籍」は、書籍もしくは書籍とみなされる物品を表すオブジェクトである。
TSV-BK オブジェクトは、`200` `201` `202` のいずれかのオブジェクト区分コードの割り当てを受けるものとする。
TSV-BK オブジェクトとしてTSVコードの割り当てを受けるオブジェクトのうち、国際標準図書番号の割り当てを受けているものは、その国際標準図書番号の桁数に応じて、13桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例又は10桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例を受けるものとする。但し、10桁と13桁の両方の割り当てを受けているものは、13桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例を受けるものとする。
システムは、国際標準図書番号の割り当てを受けている書籍オブジェクトの特例を受けないオブジェクトに対し、オブジェクト区分コード `202` を割り当てる。また、`000000000` から `999999999` までの範囲において未使用の番号のうち、任意の番号を固有番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入する。

##### 2.2.1.1. 13桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例
システムは、13桁の国際標準図書番号の割り当てを受けている書籍に対し、オブジェクト区分コード `200` を割り当てる。また、当該書籍に割り当てられた国際標準図書番号の4桁目から12桁目までの9桁の数字列を固有番号として割り当てる。但し、同一オブジェクト区分コードを有するオブジェクトの中で、当該番号が既に割り当てられている場合には、当該書籍は、13桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例を受けないオブジェクトと同様に取り扱う。

##### 2.2.1.2. 10桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例
システムは、10桁の国際標準図書番号の割り当てを受けている書籍に対し、オブジェクト区分コード `201` を割り当てる。また、当該書籍に割り当てられた国際標準図書番号の1桁目から9桁目までの9桁の数字列を固有番号として割り当てる。但し、同一オブジェクト区分コードを有するオブジェクトの中で、当該番号が既に割り当てられている場合には、当該書籍は、10桁の国際標準図書番号の割り当てを受けている書籍オブジェクトの特例を受けないオブジェクトと同様に取り扱う。

#### 2.2.2. TSV-BG オブジェクト（書籍グループ）
| オブジェクト区分 | オブジェクト区分の名称 | オブジェクト区分記号 | オブジェクト区分コード |
| :---: | :---: | :---: | :---: |
| **TSV-BG オブジェクト** | 書籍グループ | BG | 205 |

TSV-BG オブジェクト「書籍グループ」は、書籍のグループを表すオブジェクトである。
システムは、TSV-BG オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、オブジェクト区分コード `205` を割り当てる。また、`000000000` から `999999999` までの範囲において未使用の番号のうち、任意の番号を固有番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入する。

#### 2.2.3. TSV-RE オブジェクト（記録資料等）
| オブジェクト区分 | オブジェクト区分の名称 | オブジェクト区分記号 | オブジェクト区分コード |
| :---: | :---: | :---: | :---: |
| **TSV-RE オブジェクト** | 記録資料等 | RE | 210 |

TSV-RE オブジェクト「記録資料等」は、書籍以外の記録資料、文書、その他の物品を表すオブジェクトである。但し、ファイル（フラットファイル、クリアーファイル、その他専ら資料及び文書等を収納するもののことをいう。）は、その収納物の内容量が適当なものである場合に限って、単一のTSV-RE オブジェクトとしてTSVコードの割り当てを受けることができるものとする。
TSV-RE オブジェクトは、オブジェクト区分コード `210` の割り当てを受けるものとする。また、発行者別資料番号の割り当てを受けるものとする。
発行者別資料番号は、TSV-IS オブジェクト毎に分けられる4桁の連番とする。
システムは、TSV-RE オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、オブジェクト区分コード `210` を割り当てる。また、`0000` から `9999` までの範囲において、同一のTSV-IS オブジェクトに関連付けられるオブジェクトの中で、最も小さい未使用の番号を発行者別資料番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。
システムは、TSV-RE オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、当該オブジェクトに関連付けられる TSV-IS オブジェクトの発行者番号（5桁）と、発行者別資料番号（4桁）を組み合わせた9桁の数列を固有番号として割り当てる。
システムは、TSV-RE オブジェクトに対して必ず1つの TSV-IS オブジェクトを関連付ける。関連付けるTSV-IS オブジェクトを変更する必要が生じた場合は、新規のTSV-RE オブジェクトを作成する。この時、既存のTSV-RE オブジェクトは、無効化するものとし、変更記録等で関連付ける等の措置を講じる。このとき、既存のTSV-RE オブジェクトのTSVコードは、変更後も引き続き有効なものとみなし、未使用の番号には算入しない。

#### 2.2.4. TSV-IS オブジェクト（資料発行者）
| オブジェクト区分 | オブジェクト区分の名称 | オブジェクト区分記号 | オブジェクト区分コード |
| :---: | :---: | :---: | :---: |
| **TSV-IS オブジェクト** | 資料発行者 | IS | 215 |

TSV-IS オブジェクト「資料発行者」は、TSV-RE オブジェクトの発行者を表すオブジェクトである。
TSV-IS オブジェクトは、オブジェクト区分コード `215` の割り当てを受けるものとする。また、発行者番号の割り当てを受けるものとする。
発行者番号は、5桁の番号とする。
システムは、TSV-IS オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、オブジェクト区分コード `215` を割り当てる。また、`00002` から `99999` までの範囲において未使用の番号のうち、任意の番号を発行者番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。
システムは、TSV-IS オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、`0000` と、当該オブジェクトの発行者番号（5桁）を組み合わせた9桁の数列を固有番号として割り当てる。但し、システムで予約済みのTSV-IS オブジェクトに係る特例を受けるオブジェクトに対しては、当該特例に定めるところによる。
システムは、TSVコード `TSV-IS-215-0001-00000-C` 及び `TSV-IS-215-0001-00001-C` を予約し、それぞれのTSVコードを特定のオブジェクト（以下、システムで予約済みのTSV-IS オブジェクトという。）に割り当てる。特定のオブジェクトは、システムで予約済みのTSV-IS オブジェクトに係る特例に関する規定に定める。
システムは、1つ以上の TSV-RE オブジェクトと関連付けられる TSV-IS オブジェクトに対する削除操作を実行しない。
システムは、「システムで予約済みのTSV-IS オブジェクトに係る特例」を受けるオブジェクトに対する削除操作を実行しない。

##### 2.2.4.1. システムで予約済みのTSV-IS オブジェクトに係る特例（システム発行者）
システムは、自らをTSV-IS オブジェクトとして定義し、`TSV-IS-215-0001-00000-C` のTSVコードを割り当てる。
当該オブジェクトは、当該システム名等を名称とする、システムで予約済みのTSV-IS オブジェクトとする。

##### 2.2.4.2. システムで予約済みのTSV-IS オブジェクトに係る特例（デフォルト発行者）
システムは、いずれの資料発行者にも属さないTSV-RE オブジェクトに関連付けるための特例オブジェクトを定義し、`TSV-IS-215-0001-00001-C` のTSVコードを割り当てる。
当該オブジェクトは、「デフォルト」や「不明」、「共通」などを名称とする、システムで予約済みのTSV-IS オブジェクトとする。

#### 2.2.5. TSV-SH オブジェクト（書架）
| オブジェクト区分 | オブジェクト区分の名称 | オブジェクト区分記号 | オブジェクト区分コード |
| :---: | :---: | :---: | :---: |
| **TSV-SH オブジェクト** | 書架 | SH | 220 |

TSV-SH オブジェクト「書架」は、書籍や書類を収納するための物理的な構造物を表すオブジェクトである。
TSV-SH オブジェクトは、オブジェクト区分コード `220` の割り当てを受けるものとする。また、書架番号及び設置列番号の割り当てを受けるものとする。
書架番号は、7桁の番号とする。設置列番号は、書架番号毎に分けられる2桁の連番とする。
TSV-SH オブジェクトとしてTSVコードの割り当てを受けるオブジェクトのうち、既存の書架に付加されるものは、設置列番号の割り当てを受ける書架オブジェクトの特例を受けるものとする。なお、設置列番号の割り当てを受ける書架オブジェクトの特例を受けるオブジェクトは、独立したオブジェクトであるものとする。
システムは、TSV-SH オブジェクトとしてTSVコードの割り当てを受けるオブジェクトに対し、オブジェクト区分コード `220` を割り当てる。また、`0000000` から `9999999` までの範囲において未使用の番号のうち、任意の番号を書架番号として、`00` を設置列番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。
システムは、TSVコード `TSV-SH-220-0000000-XX-C` 番台（`TSV-SH-220-0000000-00-C` から `TSV-SH-220-0000000-99-C` まで）を予約し、システムで予約済みのTSV-SH オブジェクトに係る特例を受けるオブジェクトに割り当てる。特定のオブジェクトは、システムで予約済みのTSV-SH オブジェクトに係る特例に関する規定に定めるところによる。

##### 2.2.5.1. 設置列番号の割り当てを受ける書架オブジェクトの特例
システムは、既存の書架に付加される書架オブジェクトに対し、オブジェクト区分コード `220` を割り当て、付加する書架に割り当てられた書架番号を固有番号の前半7桁に割り当て、また、`01` から `99` までの範囲において、当該書架に付加されるオブジェクトの中で最も小さい未使用の番号を設置列番号として割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。

##### 2.2.5.2. システムで予約済みのTSV-SH オブジェクトに係る特例（システムデータ書架）
システムは、システムデータを収納するための特例オブジェクトを定義し、`TSV-SH-220-0000000-00-C` のTSVコードを割り当てる。
当該オブジェクトは、当該システムデータの内容を名称とする、システムで予約済みのTSV-SH オブジェクトとする。

##### 2.2.5.3. システムで予約済みのTSV-SH オブジェクトに係る特例（デフォルト書架）
システムは、いずれの書架にも属さないオブジェクトを関連付けるための特例オブジェクトを定義し、`TSV-SH-220-0000000-01-C` のTSVコードを割り当てる。
当該オブジェクトは、「デフォルト」や「未割り当て」、「共通」などを名称とする、システムで予約済みのTSV-SH オブジェクトとする。

### 2.3. オブジェクト区分とTSVコードにおける固有番号内字区切りルール

## 3. TSVコードの要件（Requirements for TSV Codes）
TSVコードは、これより先に定めるところに従って、当該コードの絶対要件を満たすものでなければならない。また、S-TSV検証コード、S-TSVコード及びS-TSV一次元シンボルは、これより先に定めるところに従って、当該コードの絶対要件を満たすものでなければならない。

### 3.1. TSVコードの絶対要件
TSVコードは、次の要件を満たすものでなければならない。
1. 当該TSVコードが完全記法に基づくものであること。
2. 当該TSVコードがオブジェクト区分ごとに定められた構成ルールに従って生成されていること。
3. 当該TSVコードが同一オブジェクト区分内で重複していないこと。
4. 当該TSVコードの上位4桁が、 `TSV-` であること。
5. 当該TSVコードのオブジェクト区分記号及びオブジェクト区分コードが、オブジェクト区分ごとに定められたものと一致していること。
6. 当該TSVコードの`2`以降の数字列が13桁であって、下位1桁が上位12桁の数字列に関するチェック数字として有効であること。

### 3.2. S-TSV検証コードの絶対要件
S-TSV検証コードは、次の要件を満たすものでなければならない。
1. 当該S-TSV検証コードが、英数字を含まない12桁の数字列であること。
2. 当該S-TSV検証コードが、対応するオブジェクトのTSVコードのオブジェクト区分コード、固有番号を適切に組み合わせて生成されていること。
3. 当該S-TSV検証コードが基づくTSVコードがチェック数字を除いて絶対要件を満たすものであること。

### 3.3. S-TSVコード・S-TSV一次元シンボルの絶対要件
S-TSVコードは、次の要件を満たすものでなければならない。また、S-TSV一次元シンボルは、シンボル全体が次の要件を満たし、かつ、次の要件を満たすS-TSVコードを内容とするものでなければならない。
1. 当該S-TSVコードが英数字を含まない13桁の数字列であること。
2. 当該S-TSVコードが、対応するオブジェクトのTSVコードのオブジェクト区分コード、固有番号、チェック数字を適切に組み合わせて生成されていること。
3. 当該S-TSVコードが基づくTSVコードが絶対要件を満たすものであること。
4. 当該S-TSVコードの下位1桁が、上位12桁の数字列に関するチェック数字として有効であること。
5. オブジェクトにS-TSV一次元シンボルを貼付等する場合は、GTIN-13（JAN-13/EAN-13）形式の一次元シンボルとして入出力されること。

### 3.4. チェック数字
TSVコードのチェック数字は、次のルールに従って算出し、使用するものとする。

1. チェック数字は、S-TSV検証コードに基づいて算出するものとする。
2. チェック数字は、TSVコード及びS-TSVコード（これに係るS-TSV一次元シンボルを含む）の全てに適用する。
3. チェック数字の算出方法を モジュール 10 ウェイト 3 と称し、算出の方法を次のとおり定める。
    1. S-TSV検証コードの12桁の数字を、左から順に1桁ずつ、奇数桁と偶数桁に分ける。ここで、最も左の桁を1桁目とする。
    2. 奇数桁の数字をすべて合計する。
    3. 偶数桁の数字をすべて合計し、その合計に3を乗じる。
    4. 奇数桁の合計と偶数桁の合計に3を乗じた値を足し合わせる。
    5. 足し合わせた値を10で割った余りを求める。
    6. 余りが0の場合はチェック数字は0とし、そうでない場合は10から余りを引いた値をチェック数字とする。
4. システムは、オブジェクトを生成する際に、当該オブジェクトの情報に基づいてTSVコードを生成するとき、上記のルールに従ってチェック数字を算出し、当該TSVコードの末尾に付与するものとする。
5. システムは、S-TSV一次元シンボルを読み取るときには、当該S-TSV一次元シンボルの示すS-TSVコードのチェック数字を検証することにより、当該S-TSV一次元シンボルの示すS-TSVコードの有効性を確認するよう努める。

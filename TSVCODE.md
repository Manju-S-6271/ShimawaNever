# TSVコード体系定義書 (TSV Code Specification)

## 0. 主文 (General Provisions)
TSVコードは、所有する物件等とこれに関連する事柄（専ら書籍や書類、その発行者、書架等。以下、これらを総じて物件という。）の識別及び管理の用に供する電磁的記録における各物件の識別子規格であって、TSVコード並びにTSVコードの子体系として制定されるS-TSVコード、S-TSV 検証コード、及びS-TSV一次元シンボルの構造、生成規則、及び運用に関する事項は、本ドキュメントに定めるところによる。
本規格は、物件に対して一意の識別子を付与することにより、物件の追跡、管理、及び相互参照を可能とすることを目的とする。
本ドキュメントは、TSVコードを用いるすべてのシステム（以下、システムという。）に対し、本ドキュメントに定めるところに従ってTSVコード、S-TSVコード、S-TSV検証コード、及びS-TSV一次元シンボルを生成し、管理し、運用することを求める。

## 1. システムの要件（Requirements for System）
システムは、次の要件を満たす。

1. TSVコードを割り当てる機能を有すること。
2. 適切なデータベースを構築し、またはこれらにアクセスする機能を有すること。
3. データベース上において、TSVコードを識別子とする物件に係るオブジェクトデータを作成し、TSVコードによって参照し、更新し、削除する機能を有すること。
4. S-TSV一次元シンボルの発行を行う機能を有し、またはこのために必要な情報を出力する機能を有する場合は、TSVコードをS-TSVコードに変換し、S-TSVコードをTSVコードに変換する機能を有すること。
5. S-TSV一次元シンボルの発行を行う機能を有し、またはこのために必要な情報を出力する機能を有する場合は、当該システムがS-TSVコードでないGS1不特定識別番号を内容とするGTIN-13（JAN標準タイプ［JAN-13］/EAN-13）の取り扱いを行っていないこと。

## 2. コード構造 (Code Structure)
TSVコードは、これより先に示す構成要素からなるコード体系である。

### 2.1. オブジェクト
システムは、TSVコードによって識別されるそれぞれの物件を、1つのオブジェクトとして取り扱う。
TSVコードの割り当てを受けるオブジェクトは、次の番号及び記号の割り当てを受けなければならない。また、これらの番号及び記号は、本ドキュメントに定めるところに従って割り当てる。

1. オブジェクト区分記号
2. オブジェクト区分コード
3. 固有番号

### 2.2. 完全記法 TSVコード (Full Notation / TSV Code)
TSVコードは、18桁の英数字及びハイフンで構成される文字列であって、本コード体系における完全記法の識別子である。
システムは、各オブジェクトに対して本記法に基づくTSVコードを生成し、これらを主たる識別子として割り当てる。
TSVコードの構造は次に示すとおりである。

- **フォーマット:** `TSV-[TT]-[2SS]-[XXXXXXXXX]-[C]`
- **構成:**
    - `TSV`: システム識別子（固定値）
    - `TT`: オブジェクト区分記号（2文字）
    - `2SS`: オブジェクト区分コード（3桁）
    - `XXXXXXXXX`: 固有番号（字区切りを除いて9桁）
    - `C`: チェック数字（1桁）

### 2.3. 検証用数字 S-TSV検証コード (Validation Number / S-TSV Validation Code)
S-TSV検証コードは、TSVコード及びS-TSVコードに関するチェック数字の算出の用に供する12桁の数字列である。
システムは、オブジェクトに対してTSVコードを割り当てるとき、本記法に基づくS-TSV検証コードを生成し、当該コードによって当該オブジェクトのTSVコードのチェック数字を算出し、当該チェック数字を当該オブジェクトのTSVコードの末尾に付与する。
S-TSV検証コードの構造は次に示すとおりである。

- **フォーマット:** `[2SS][XXXXXXXXX]`（計12桁）
- **構成:** オブジェクト区分コード(3) + 固有番号(9)

### 2.4. 短縮記法 S-TSVコード (Short Notation / S-TSV Code)
S-TSVコードは、13桁の数字列であって、本コード体系におけるオブジェクトに割り当てられるGS1不特定識別番号である。
システムは、貼付用S-TSV一次元シンボルを発行するとき、当該オブジェクトに対して本記法に基づくS-TSVコードを生成する。但し、本ドキュメントは、S-TSVコードのデータベース内保管を強制するものではない。
S-TSVコードの構造は次に示すとおりである。

- **フォーマット:** `[2SS][XXXXXXXXX][C]`（計13桁）
- **構成:** オブジェクト区分コード(3) + 固有番号(9) + チェック数字(1)

### 2.5. 短縮記法に基づく管理コード S-TSV一次元シンボル (Short Notation-based Management Code / S-TSV One-Dimensional Symbol)
S-TSV一次元シンボルは、本コード体系によって管理されるオブジェクトに貼付されるGTIN-13（JAN標準タイプ［JAN-13］/EAN-13）形式の一次元シンボルである。
システムは、物件に対する管理コードの発行の請求を受けたとき、当該オブジェクトのS-TSVコードを内容とするS-TSV一次元シンボルを生成し、当該S-TSV一次元シンボルを当該物件に貼付するための管理コードとして発行する。

## 3. オブジェクト区分の設置（Placement of Object Categories）
システムは、オブジェクト区分を設置する。
設置するオブジェクト区分は、設置するシステムごとに定義する。
システムは、オブジェクト区分ごとに、オブジェクト区分記号及びオブジェクト区分コードを割り当てる。
システムは、オブジェクト区分ごとに、当該オブジェクト区分に属するオブジェクトの固有番号の構成ルール（固有番号の生成規則及び字区切りの規則を含む。）を定める。
システムがオブジェクト区分に定めるオブジェクト区分記号及びオブジェクト区分コードの条件は、次に定める。

1. オブジェクト区分記号は、英大文字2文字で構成されること。
2. オブジェクト区分コードは、200から299までの3桁の数字で構成されること。
3. 割り当てられるオブジェクト区分記号及びオブジェクト区分コードが、同一システム内で重複していないこと。

## 4. TSVコードの要件（Requirements for TSV Codes）
TSVコードは、これより先に定めるところに従って、当該コードの絶対要件を満たすものでなければならない。また、S-TSV検証コード、S-TSVコード及びS-TSV一次元シンボルは、これより先に定めるところに従って、当該コードの絶対要件を満たすものでなければならない。

### 4.1. TSVコードの絶対要件
TSVコードの絶対要件は、次に定める。

1. 当該TSVコードが完全記法に基づくものであること。
2. 当該TSVコードがオブジェクト区分ごとに定められた構成ルールに従って生成されていること。
3. 当該TSVコードが同一オブジェクト区分内で重複していないこと。
4. 当該TSVコードの上位4桁が、 `TSV-` であること。
5. 当該TSVコードのオブジェクト区分記号及びオブジェクト区分コードが、オブジェクト区分ごとに定められたものと一致していること。
6. 当該TSVコードの`2`以降の数字列が13桁であって、下位1桁が上位12桁の数字列に関するチェック数字として有効であること。
7. 固有番号が数字及びハイフンのみで構成されていること。

### 4.2. S-TSV検証コードの絶対要件
S-TSV検証コードの絶対要件は、次に定める。

1. 当該S-TSV検証コードが、英数字を含まない12桁の数字列であること。
2. 当該S-TSV検証コードが、対応するオブジェクトのTSVコードのオブジェクト区分コード、固有番号を適切に組み合わせて生成されていること。
3. 当該S-TSV検証コードが基づくTSVコードがチェック数字を除いて絶対要件を満たすものであること。

### 4.3. S-TSVコード・S-TSV一次元シンボルの絶対要件
S-TSVコードの絶対要件は、次に定める。S-TSV一次元シンボルは、その内容とするS-TSVコードが次の絶対要件を満たすものであることを要件とする。

1. 当該S-TSVコードが英数字を含まない13桁の数字列であること。
2. 当該S-TSVコードが、対応するオブジェクトのTSVコードのオブジェクト区分コード、固有番号、チェック数字を適切に組み合わせて生成されていること。
3. 当該S-TSVコードが基づくTSVコードが絶対要件を満たすものであること。
4. 当該S-TSVコードの下位1桁が、上位12桁の数字列に関するチェック数字として有効であること。
5. オブジェクトにS-TSV一次元シンボルを貼付等する場合は、GTIN-13（JAN標準タイプ［JAN-13］/EAN-13）形式の一次元シンボルとして入出力されること。

### 4.4. チェック数字
TSVコードのチェック数字の要件は、次に定める。

1. チェック数字は、S-TSV検証コードに基づいて算出するものとする。
2. チェック数字は、TSVコード及びS-TSVコード（これに係るS-TSV一次元シンボルを含む）の全てに適用する。
3. チェック数字の算出方法を モジュール 10 ウェイト 3 と称し、算出の方法を次のとおり定める。
    1. S-TSV検証コードの12桁の数字を、左から順に1桁ずつ、奇数桁と偶数桁に分ける。ここで、最も左の桁を1桁目とする。
    2. 奇数桁の数字をすべて合計する。
    3. 偶数桁の数字をすべて合計し、その合計に3を乗じる。
    4. 奇数桁の合計と偶数桁の合計に3を乗じた値を足し合わせる。
    5. 足し合わせた値を10で割った余りを求める。
    6. 余りが0の場合はチェック数字は0とし、そうでない場合は10から余りを引いた値をチェック数字とする。
4. システムは、オブジェクトを生成する際に、当該オブジェクトの情報に基づいてTSVコードを生成するとき、上記のルールに従ってチェック数字を算出し、当該TSVコードの末尾に付与するものとする。
5. システムは、S-TSV一次元シンボルを読み取るときには、当該S-TSV一次元シンボルの示すS-TSVコードのチェック数字を検証することにより、当該S-TSV一次元シンボルの示すS-TSVコードの有効性を確認するよう努める。

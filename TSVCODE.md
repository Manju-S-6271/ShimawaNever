# TSVコード体系定義書 (TSV Code Specification)

TSVコードとは、主としてWebアプリケーション「ShimawaNever」で使用される、書籍、書類、発行者、および書架オブジェクトを体系的に管理するための識別子規格であって、その構造は、本ドキュメントに定めるところによる。

## 1. コード構造 (Code Structure)

TSVコードは、表示方法により「完全記法」と「短縮記法」の2種類の形式を持つ。

### 1.1. 完全記法 TSVコード (Full Notation / TSV Code)
完全形式の識別子。TSVコードを用いるシステム内において、各オブジェクトは本記法に基づくTSVコードを主たる識別子として有さなければならない。
英数字及び4つから6つまでのハイフンで構成される文字列であって、その構造は次に示すとおりである。

- **フォーマット:** `TSV-[TT]-[2SS]-[XXXXXXXXX]-[C]`
- **構成:**
    - `TSV`: システム識別子（固定値）
    - `TT`: オブジェクト区分記号（2文字）
    - `2SS`: オブジェクト区分コード（3桁）
    - `XXXXXXXXX`: 固有番号（9桁）
    - `C`: チェック数字（1桁）

### 1.2. 短縮記法 S-TSVコード (Short Notation / S-TSV Code)
短縮形式の識別子。後述の「S-TSV 一次元シンボル」で用いる識別子であって、システム内においては、完全記法のTSVコードと対応付けられるものでなければならない。
英数字を含まない13桁の数字列（ハイフンを含まない）であって、その内容は各オブジェクトのTSVコードに準ずるものとし、その構造は次に示すとおりである。

- **フォーマット:** `[2SS][XXXXXXXXX][C]`（計13桁）
- **構成:** オブジェクト区分コード(3) + 固有番号(9) + チェック数字(1)

### 1.3. 短縮記法に基づく管理コード S-TSV一次元シンボル (Short Notation-based Management Code / S-TSV One-Dimensional Symbol)
S-TSV コードを内容とするGTIN-13（JAN-13/EAN-13）形式一次元シンボルとして入出力する際に使用される形式とする。

## 2. オブジェクト別定義 (Object Definitions)

### 2.1. オブジェクト区分
TSVコードの割り当てを受けるオブジェクトは、次に示すいずれかのオブジェクト区分に属するものとする。
システムは、オブジェクト区分ごとに定められたコード体系に従って、TSVコードを生成しなければならない。また、システムは、いずれのオブジェクト区分にも属さないオブジェクトに対して、TSVコードを割り当ててはならない。

| オブジェクト区分 | 名称 | 説明 |
| :--- | :--- | :--- |
| **TSV-BK オブジェクト** | 書籍 | オブジェクトのうち、専ら書籍としての性質を有するもの。 |
| **TSV-BG オブジェクト** | 書籍グループ | TSV-BK オブジェクトをグループ化するためのオブジェクト。書籍のシリーズ（前後編に分されるもの、定期刊行物、雑誌扱いのうち複数巻に分かれるものなど）を表す。 |
| **TSV-RE オブジェクト** | 記録資料等 | オブジェクトのうち、書籍以外の記録資料、文書、その他の物品を表すもの。 |
| **TSV-IS オブジェクト** | 資料発行者 | TSV-RE オブジェクトの発行者を表すオブジェクト。TSV-RE オブジェクトは、必ず1つの TSV-IS オブジェクトと関連付けられる。 |
| **TSV-SH オブジェクト** | 書架 | 書籍や書類を収納するための物理的な構造物を表すオブジェクト。書架は、親子構造を持つことができる。 |

### 2.2. オブジェクト区分とオブジェクト区分記号及びオブジェクト区分コード
TSVコードの割り当てを受けるオブジェクトは、そのオブジェクト区分に対応するオブジェクト区分記号及びオブジェクト区分コードの割り当てを受けるものとする。
オブジェクト区分別のオブジェクト区分記号及びオブジェクト区分コードは、次条以降に定めるところによる。

### 2.3. オブジェクト区分と固有番号の構成ルール
オブジェクト区分の定義は、本項以降に定めるところによる。
システムは、オブジェクト区分ごとに定められた構成ルールに従って、固有番号を生成しなければならない。

#### 2.3.1. TSV-BK オブジェクト（書籍）
TSV-BK オブジェクト「書籍」は、書籍もしくは書籍とみなされる物品を表すオブジェクトである。
TSV-BK オブジェクトは、`200` もしくは `201` のオブジェクト区分コードの割り当てを受けるものとする。

TSV-BK オブジェクトとしてTSVコードの割り当てを受けるオブジェクトのうち、ISBNの割り当てを受けているものは、ISBNの割り当てを受けている書籍の特例を受けるものとする。但し、ISBNの割り当てを受けている書籍の特例の適用外となるオブジェクトは、ISBNの割り当てを受けている書籍の特例を受けないオブジェクトと同様に取り扱う。
ISBNの割り当てを受けている書籍の特例を受けないオブジェクトは、オブジェクト区分コード `201` の割り当てを受け、その固有番号に、`000000000`を初期値とする連番であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てる。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。

##### 2.3.1.1. ISBNの割り当てを受けている書籍の特例
ISBNの割り当てを受けている書籍は、オブジェクト区分コード `200` の割り当てを受け、その固有番号は、当該書籍に割り当てられたISBNの4桁目から12桁目までの9桁の数列を準用してこの割り当てを受けるものとする。ISBN内のチェック数字は、これを除外して割り当て、別途TSVコードのチェック数字を算出するものとする。
但し、同一TSVコードの割り当てを受けたオブジェクトが既に存在する場合には、当該書籍は、本特例を受けることができない。

#### 2.3.2. TSV-BG オブジェクト（書籍グループ）
TSV-BG オブジェクト「書籍グループ」は、書籍のグループを表すオブジェクトである。
TSV-BG オブジェクトは、オブジェクト区分コード `202` の割り当てを受けるものとする。
TSV-BG オブジェクトの固有番号は、`000000000`を初期値とする連番であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。

#### 2.3.3. TSV-RE オブジェクト（記録資料等）
TSV-RE オブジェクト「記録資料等」は、書籍以外の記録資料、文書、その他の物品を表すオブジェクトである。但し、ファイル（フラットファイル、クリアーファイル、その他専ら資料及び文書等を収納するもののことをいう。）は、その収納物の内容量が適当なものである場合に限って、単一のTSV-RE オブジェクトとしてTSVコードの割り当てを受けることができるものとする。
TSV-RE オブジェクトは、オブジェクト区分コード `210` の割り当てを受けるものとする。
TSV-RE オブジェクトの固有番号は、当該オブジェクトに関連付けられる TSV-IS オブジェクトの発行者番号下位5桁（5桁）と、当該発行者内での書類連番（4桁）を組み合わせた9桁の数列であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。
TSV-RE オブジェクトは、必ず1つの TSV-IS オブジェクトと関連付けられるものとする。関連付けるTSV-IS オブジェクトを変更する必要が生じた場合は、新規のTSV-RE オブジェクトを作成するものとする。この時、既存のTSV-RE オブジェクトは、無効化するものとし、変更記録等で関連付ける等の措置を講じるものとする。この時、既存のTSV-RE オブジェクトのTSVコードは、変更後も引き続き有効なものとみなし、未使用の番号に算入しないものとする。

#### 2.3.4. TSV-IS オブジェクト（資料発行者）
TSV-IS オブジェクト「資料発行者」は、TSV-RE オブジェクトの発行者を表すオブジェクトである。
TSV-IS オブジェクトは、オブジェクト区分コード `211` の割り当てを受けるものとする。
TSV-IS オブジェクトの固有番号は、固定値 `0000` を上位4桁に、`00002`を初期値とする連番の発行者番号を下位5桁に組み合わせた9桁の数列であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。また、システムで予約済みのTSV-IS オブジェクトに係る特例を受けるオブジェクトは、この規定に優越して、当該特例に定めるところによる固有番号の割り当てを受けるものとする。
システムは、TSVコード `TSV-IS-211-0001-00000-3` 及び `TSV-IS-211-0001-00001-0` を予約し、それぞれのTSVコードを特定のオブジェクト（以下、システムで予約済みのTSV-IS オブジェクトという。）に割り当てるものとする。特定のオブジェクトは、システムで予約済みのTSV-IS オブジェクトに係る特例に関する規定に定める。
システムは、1つ以上の TSV-RE オブジェクトと関連付けられる TSV-IS オブジェクトに対する削除操作を実行してはならない。
システムは、「システムで予約済みのTSV-IS オブジェクトに係る特例」を受けるオブジェクトに対する削除操作を実行してはならない。

##### 2.3.4.1 システムで予約済みのTSV-IS オブジェクトに係る特例（システム）
システムは、自らをTSV-IS オブジェクトとして定義し、`TSV-IS-211-0001-00000-3` のTSVコードを割り当てるものとする。
当該オブジェクトは、当該システム名等を名称とする、システムで予約済みのTSV-IS オブジェクトとする。

##### 2.3.4.2 システムで予約済みのTSV-IS オブジェクトに係る特例（デフォルト）
システムは、いずれの資料発行者にも属さないTSV-RE オブジェクトに関連付けるための特例オブジェクトを定義し、`TSV-IS-211-0001-00001-0` のTSVコードを割り当てるものとする。
当該オブジェクトは、「デフォルト」や「不明」、「共通」などを名称とする、システムで予約済みのTSV-IS オブジェクトとする。

#### 2.3.5. TSV-SH オブジェクト（書架）
TSV-SH オブジェクト「書架」は、書籍や書類を収納するための物理的な構造物を表すオブジェクトである。
TSV-SH オブジェクトは、オブジェクト区分コード `220` の割り当てを受けるものとする。
TSV-SH オブジェクトの固有番号は、書架番号（7桁）と子オブジェクト番号（2桁）を組み合わせた9桁の数列であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。但し、欠番は、割り当てられたオブジェクトがデータベース上から完全に消去されている場合に限って、未使用の番号に算入するものとする。
書架番号は、`0000000`を初期値とする連番であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。子オブジェクト番号は、親オブジェクトの書架番号を継承し、`00`を初期値とする連番であって、同一オブジェクト区分コードを有するオブジェクトの中で、最も小さい未使用の番号を割り当てるものとする。親オブジェクトの子オブジェクト番号は `00` とし、子オブジェクトの子オブジェクト番号は `01` から `99` とするものとする。
それぞれの子オブジェクトは独立したオブジェクトとして取り扱う。
システムは、子オブジェクトが存在するオブジェクトに対する削除操作を実行してはならない。

### 2.4. オブジェクト区分とTSVコードにおける固有番号内字区切りルール
TSVコードの割り当てを受けるオブジェクトは、TSVコードの表示において、固有番号をオブジェクト区分ごとに定められた字区切りルールに従いハイフンで区切るものとする。
オブジェクト区分ごとの字区切りルールは、次のとおり定める。

| オブジェクト区分 | （特例を受ける場合はその特例の名称） | 固有番号内字区切りルール | 凡例 |
| :--- | :--- | :--- | :--- |
| **TSV-BK** | | 3桁ごとに区切る。 | `TSV-BK-200-000-000-000-0` |
| **TSV-BK** | ISBNの割り当てを受けている書籍の特例 | ISBNコードに準ずる。 | `TSV-BK-200-4-01-000001-0` |
| **TSV-BG** | | 3桁ごとに区切る。 | `TSV-BG-202-000-000-000-0` |
| **TSV-RE** | | 発行者番号と書類番号を区切る。 | `TSV-RE-210-00001-0001-0` |
| **TSV-IS** | | 固定値と発行者番号を区切る。 | `TSV-IS-211-0000-00001-0` |
| **TSV-IS** | システムで予約済みのTSV-IS オブジェクトに係る特例 | 4桁目後で区切る。 | `TSV-IS-211-0001-00000-3` |
| **TSV-SH** | | 書架番号と子オブジェクト番号を区切る。 | `TSV-SH-220-0000001-00-9` |

## 3. TSVコードの構成ルール (TSV Code Construction Rules)

### 3.1. TSVコードの絶対要件
TSVコードは、次の要件を満たすものでなければならない。
1. 当該TSVコードが完全記法に基づくものであること。
2. 当該TSVコードがオブジェクト区分ごとに定められた構成ルールに従って生成されていること。
3. 当該TSVコードが同一オブジェクト区分内で重複していないこと。
4. 当該TSVコードの上位4桁が、 `TSV-` であること。
5. 当該TSVコードのオブジェクト区分記号及びオブジェクト区分コードが、オブジェクト区分ごとに定められたものと一致していること。

### 3.2. S-TSVコード・S-TSV一次元シンボルの絶対要件
S-TSVコード及びS-TSV一次元シンボルは、次の要件を満たすものでなければならない。
1. 当該S-TSVコードが英数字を含まない13桁の数字列を内容とするGTIN-13（JAN-13/EAN-13）形式の一次元シンボルであること。但し、システム上で取り扱う場合においては、13桁の数字列であること。
2. 当該S-TSVコードが、対応するオブジェクトのTSVコードのオブジェクト区分コード、固有番号、チェック数字を適切に組み合わせて生成されていること。
3. 当該S-TSVコードが基づくTSVコードが前条の絶対要件を満たすものであること。
4. オブジェクトにS-TSV一次元シンボルを貼付等する場合は、GTIN-13（JAN-13/EAN-13）形式の一次元シンボルとして入出力されること。

### 3.3. チェック数字の算出ルール
TSVコードのチェック数字は、次のルールに従って算出されるものとする。

1. チェック数字は、S-TSVコードの上位12桁の数字を基に算出し、TSVコード及びS-TSVコードの末尾に付与するものとする。
2. チェック数字の算出方法を モジュール 10 ウェイト 3 と称し、算出の方法を次のとおり定める。
    1. S-TSVコードの上位12桁の数字を左から順に、奇数桁と偶数桁に分ける。
    2. 奇数桁の数字をすべて合計する。
    3. 偶数桁の数字をすべて合計し、その合計に3を乗じる。
    4. 奇数桁の合計と偶数桁の合計に3を乗じた値を足し合わせる。
    5. 足し合わせた値を10で割った余りを求める。
    6. 余りが0の場合はチェック数字は0とし、そうでない場合は10から余りを引いた値をチェック数字とする。
3. 「ISBNの割り当てを受けている書籍の特例」を受けるTSV-BK オブジェクトを含め例外を認めない。


| オブジェクト区分 | システム識別番号 (AAA) | 固有番号 (BBBBBBBBB) の構成ルール | 備考 |
| :--- | :---: | :--- | :--- |
| **TSV-BK** | 200 | ISBNの4〜12桁目（9桁）を割り当て | ISBN保有書籍 |
| **TSV-BK** | 201 | 9桁の連番 | ISBN非保有書籍 |
| **TSV-BG** | 202 | 9桁の連番 | 書籍グループ |
| **TSV-RE** | 210 | 発行者番号(5桁) + 書類番号(4桁) | 書類 |
| **TSV-IS** | 211 | 固定値`0000`(4桁) + 発行者番号(5桁) | 書類発行者 |
| **TSV-SH** | 220 | 書架番号(7桁) + 子オブジェクト番号(2桁) | 書架 |

## 3. 詳細採番ルール

### 3.1 TSV-RE (書類) と TSV-IS (発行者) のリレーション
- **TSV-IS:** 固有番号の下5桁に発行者番号（連番）を格納する。上位4桁は将来の拡張用として `0` で埋める。
- **TSV-RE:** 固有番号の上5桁にTSV-ISの発行者番号をコピーし、下4桁にその発行者内での書類連番を付与する。

### 3.2 TSV-SH (書架) の親子構造
- **親オブジェクト:** 子オブジェクト番号（末尾2桁）を `00` とする。
- **子オブジェクト:** 親の書架番号（前7桁）を継承し、末尾2桁に `01`〜`99` の連番を付与する。
